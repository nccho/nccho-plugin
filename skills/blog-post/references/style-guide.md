# nclovehs.blogspot.com 문체 가이드

2026-07-01에 실제 게시글을 분석해 정리한 카테고리별 문체 프로필. 새 글을 쓸 때 참고용 기준이며, 시간이 지나 문체가 바뀌었다면 최신 글을 다시 확인해 이 문서를 갱신한다.

## A. 프로그래밍/기술 (참고 글: "Git Bash 한글 깨짐 현상, 5분 만에 해결하기")

**제목 형식**: `[카테고리(플랫폼)] 문제/주제 + 이모지` 예) `[윈도우(Windows)] Git Bash 한글 깨짐 현상, 5분 만에 해결하기 (명령어 총정리)`

**전체 구조**
1. 문제 제시 + 해결책 예고 (인트로)
2. 원인 설명 섹션 (🤔 왜 ~일까?)
3. 해결 방법 섹션, 단계별 (🚀 1단계, 2단계...)
4. 코드블럭 (명령어 나열)
5. 태그

**어투**: 존댓말(합니다체) 기반, 친근한 톤. 예) "겪게 되는 불편한 순간이 있습니다", "알려드리겠습니다", "답답한 ~문제", "쾌적한 ~환경"

**이모지**: 소제목 앞에 배치. 🤔(원인), 🚀(해결 시작), ✅(완료/테스트), ⚙️(설정) 등 상황에 맞게.

**코드블럭**: 언어 지정(bash 등) 후 명령어 제시. 각 줄마다 `# 번호. 설명` 형식의 주석을 단다.

**마무리**: 요약 + 다음 글 예고 정도의 짧은 인사. 태그 나열로 종료.

## B. 여행 (참고 글: 교토 당일치기, 오사카 USJ/호텔, 일본 여행 체크리스트 — 사용자가 직접 쓴 여행 글은 일본 여행 글 기준)

**제목 형식**: `[지역/코스명] 핵심 내용` 예) `[교토 당일치기] 난바 출발 은각사 & 청수사 완벽 코스`

**전체 구조**
1. 총평 & 요약
2. 상세 코스/일정 (장소 순서대로)
3. 놓치면 후회할 먹을거리
4. 포토 스팟/팁
5. 함께 보면 좋은 글 (시리즈 다른 글 링크)

**어투**: 존댓말/반말 혼합. 설명은 존댓말("관광합니다", "하차합니다"), 팁·감탄은 반말 섞어 사용("사람이 많으니 주의 하세요", "괜히 일본 국보가 아닙니다").

**정보 표기**
- 장소: `**[장소명](구글맵 링크)**` 형식
- 이동수단: 화살표(`-->`)로 경로 표시
- 가격: "1개에 150엔", "총 15,000 ¥"처럼 통화 단위 명기
- 이모지로 섹션 구분 (음식 소개 시 음식 이모지 🍢🥩🍡 등)

**인사 패턴**: 서두 인사 없이 바로 본문(총평)으로 시작. 마무리도 별도 인사 없이 태그/관련 글 링크로 종료.

## C. 맛집 리뷰 (참고 글: 제천 서민막국수 리뷰)

**제목 형식**: `[지역 맛집] 핵심 특징 - 상호명` 예) `[제천 맛집] 예약 없이 바로 입장! 메밀 90% 순메밀 막국수 - 서민막국수 제천본점`

**전체 구조**
1. 총평 & 요약
2. 메뉴 & 가격
3. 방문 정보 (위치/영업시간/주차/전화 등)

**어투**: 존댓말 기반, 감정 표현이 풍부. 예) "메밀 향이 확실하게 납니다", "대만족", "재방문 의사 200%", "기분 좋게 마무리했습니다. 😄"

**정보 표기**
- 메뉴: `이모지 + 메뉴명 - 가격` 형식. 예) "🍜 순메밀 물막국수 - 12,000원"
- 방문정보: `기호 + 항목명: 내용` 구조. 📍위치 🕐영업시간 🅿️주차 📞전화 ✅팁

**인사 패턴**: 서두/마무리 인사 없음. 실용 정보와 감정 평가를 바로 이어 붙인다.

기존 슬러그는 한글 제목을 음역하지 않고, **검색 유입을 노린 영문 키워드/에러 메시지/지명·음식명 조합**으로 짧게 만든다.

- `jecheon-seomin-makguksu-review`
- `kyoto-day-trip-ginkakuji-kiyomizudera-course`

150자 이내, 글에서 실제로 다루는 핵심 키워드(에러 메시지, 지명, 음식명, 가격 등)를 그대로 넣어 검색 노출을 높인다.
