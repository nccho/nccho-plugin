# Naver Place Summarizer

네이버 지도/플레이스 링크에서 장소 정보를 뽑아 **블로그 맛집 글 소재** 형태로 요약한다.
네이버는 `WebFetch`가 전 도메인 차단되므로 `fetch-blocked` 스킬의 curl 폴백 기법을 사용한다. 추출은 무거운 HTML 파싱이라 **subagent에 위임**해 메인 대화를 깨끗하게 유지한다.

## 1. 링크 확인

사용자가 준 것이 `naver.me` 단축링크, `map.naver.com/...`, `m.place.naver.com/place/{id}/...`, 혹은 숫자 `place_id` 중 무엇이든 그대로 다음 단계로 넘긴다. 스크립트가 알아서 `place_id`를 뽑는다(단축링크는 리다이렉트를 따라간다).

## 2. 추출 (subagent 위임)

`searcher`(또는 general-purpose) subagent에게 아래를 시킨다. **원본 HTML을 반환하지 말고 스크립트가 출력한 JSON만** 받아온다.

```
python3 <플러그인경로>/skills/naver-place/scripts/extract_naver_place.py "<링크 또는 place_id>"
```

- 스크립트가 `m.place.naver.com`의 home/review 페이지를 브라우저 헤더 curl로 받아 `window.__APOLLO_STATE__` JSON을 파싱하고, 아래 필드를 담은 JSON 하나를 stdout으로 낸다:
  `base`(상호/분류/주소/전화/좌표), `menus`(이름·가격·설명), `hours`(요일별 영업시간 + `breaks` 브레이크 타임), `status`, `review_count_theme`(테마별 언급 수), `booking`(예약 URL 단서), `reviews`(대표 리뷰 본문), `url`, `place_id`.
- 실패(빈 `base`)하면 `fetch-blocked` 스킬 절차로 폴백하거나, 사용자에게 페이지 접속이 되는지 확인한다.

## 3. 요약 정리 (출력 형식 고정)

받은 JSON을 아래 항목으로 정리한다. 블로그 글 뼈대와 1:1로 대응되도록 순서를 지킨다.

```
🏷️ 기본 정보
- 상호 / 분류 / 도로명·지번 주소 / 전화 / 영업시간(hours + status) / 좌표
- **브레이크 타임**: `hours[].breaks`에 값이 있으면(주로 식당) "브레이크 15:00~17:00"처럼 함께 표기한다. 비어 있으면 생략하고, 식당인데 정보가 없으면 "브레이크 타임 정보 없음"으로 밝힌다.

🍽️ 대표 시그니처 메뉴  ← menus 상위 3~5개, 가격 포함
- (가장 위에 있거나 '세트'·'시그니처' 명칭 우선)

📅 예약 가능 방법  ← booking 필드 기준
- bookingUrl/네이버예약 단서 있으면 "네이버 예약 가능" + 링크
- 없으면 전화(phone)로 "전화 문의", 그것도 애매하면 "현장 방문(정보 없음)"으로 정직하게

🅿️ 주차  ← review_count_theme의 '주차' 카운트 + reviews 본문의 '주차' 언급
- 명시 필드가 없으므로 리뷰 근거로 추론하고, 근거가 약하면 "리뷰상 언급 기준"이라고 밝힌다

✨ 그 장소만의 특색  ← themes 상위 + reviews에서 반복되는 포인트
- 예: 한옥 감성, 체험거리, 뷰, 특산물 활용 등 반복 키워드를 묶어 2~4개

📝 그 외 정보
- 방문자/사진 리뷰 규모(테마 카운트로 대략), 대표 리뷰 2~3개 짧게 발췌
```

### 주의

- **네이버는 2021년에 별점을 폐지**했다(`avgRating:0`). 별점을 지어내지 말고, 대신 `review_count_theme`(맛/만족도/분위기 등 언급 수)를 만족도 지표로 쓴다.
- 가격·영업시간·전화는 "방문 시점 기준, 변동 가능" 단서를 붙인다.
- 리뷰 발췌 시 특정 개인을 식별할 수 있는 내용은 옮기지 않는다.

## 4. 블로그 연계 (blog-post)

정리 끝에, 이 요약을 **`blog-post` 스킬의 맛집 카테고리 소재로 넘길지** 물어본다. 넘길 때는 위 6개 블록을 그대로 전달하면 맛집 글 구조(총평 → 메뉴 → 정보 → 특색 → 태그)로 바로 이어진다. 개인정보(전화번호 등)는 blog-post 규칙에 따라 본문에서 다듬는다.
