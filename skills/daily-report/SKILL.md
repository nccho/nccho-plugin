# Daily Report Writer

회사 일일 업무보고서를 작성하고, `~/.claude/worklog/` 아래 날짜별 markdown 파일에 누적 보관한다.
작성 문체·구조는 반드시 `references/style-guide.md`를 따른다 (두괄식 · 성과 중심 · 서술형 개조식 · CLEAR 장애물 보고 등).

## 1. 입력 방식 판별

사용자 요청에서 아래 셋 중 하나를 판별한다. 모호하면 물어본다.

1. **직접 입력**: 사용자가 오늘 한 작업 내용을 텍스트로 준다 → 그대로 보고 내용의 원천으로 삼는다.
2. **문서 링크**: 작업 내용이 담긴 문서 URL을 준다 → `WebFetch`로 문서를 읽고 분석해 보고 내용을 뽑는다. (사내 접근 불가 링크면 사용자에게 내용을 붙여달라고 요청)
3. **세션 기반**: "이번 세션 기반으로", "오늘 여기서 한 작업으로" 등 → 현재 Claude Code 세션에서 실제로 수행한 작업(수정 파일, 해결한 문제, 실행 결과)을 근거로 보고 내용을 구성한다.

## 2. 메타데이터 수집 (통계용)

각 보고 항목에 대해 아래를 확보한다. 없으면 사용자에게 짧게 묻는다.

- **작업 분야**: 프로젝트명 또는 게임명 (예: `프로젝트A`, `OO게임 라이브`)
- **소요 시간**: 시간 단위 (예: `2.5h`). 세션 기반이면 추정치를 제안하고 사용자 확인을 받는다.

하나의 하루에 서로 다른 분야/작업이 여러 개일 수 있다. 각각을 별도 항목으로 기록한다.

## 3. 보고 내용 작성

`references/style-guide.md`의 규칙과 출력 템플릿을 그대로 적용한다. 핵심만 재확인:

- 결론 한 줄을 맨 앞에 (두괄식)
- 활동 나열이 아니라 성과·효과 중심 (가능하면 수치)
- 개조식(`~함/~임`) 기본, 배경·맥락만 짧게 서술
- 진행 중인 큰 작업은 `전체 N단계 중 M단계 · 약 %` + 완료/남은 일
- 장애물은 CLEAR(원인-시도-영향-요청-후속)로 압축
- 일정은 조기·해결중심·무비난

## 4. 파일 저장 (날짜별 누적)

- 저장 폴더: `~/.claude/worklog/` (없으면 생성)
- 파일명: 오늘 날짜 `YYYY-MM-DD.md` (사용자가 다른 날짜를 지정하면 그 날짜)
- **덮어쓰지 않는다 (append-only)**: 파일이 이미 있으면 기존 내용을 유지하고 새 항목을 **맨 끝에 이어붙인다**. 여러 Claude Code 세션에서 같은 날 파일에 기록할 수 있으므로, 저장 전 반드시 기존 파일을 먼저 읽고 append 한다.
- 파일이 없으면 날짜 제목(`# YYYY-MM-DD 일일 업무 로그`)으로 새로 만든다.

### 항목 형식 (통계 파싱 가능하게 고정)

각 작업 항목은 아래 heading 형식으로 시작한다. `분야`/`소요` 토큰은 나중에 스캔·집계할 수 있도록 형식을 고정한다.

```markdown
## HH:MM · 분야: <프로젝트/게임> · 소요: <X>h

[결론 한 줄 요약]

■ 오늘 완료
- ...

(해당 시 ■ 진행 중 / ■ 장애물·요청 / ■ 일정)
```

여러 항목을 이어붙일 때 항목 사이는 빈 줄로 구분한다 (`---` 구분선은 넣지 않아도 됨).

## 5. Markdown 문법 검사 (필수)

파일을 저장한 뒤 **반드시** 생성/수정된 markdown 파일의 문법을 검사한다. 통과하지 못하면 고쳐서 다시 저장한다.

우선순위대로 사용 가능한 도구를 쓴다:

1. `markdownlint`가 있으면: `npx --no-install markdownlint-cli "<파일>"` 또는 `markdownlint "<파일>"`
2. 없으면 최소 검사를 직접 수행:
   - 코드펜스(```)의 개수가 짝수인지 (열고 안 닫힌 블록 없음)
   - heading `#` 뒤 공백, 리스트 마커 뒤 공백 등 기본 문법
   - 파일이 유효한 UTF-8 텍스트로 다시 읽히는지

검사 결과(사용한 도구, 통과 여부)를 사용자에게 한 줄로 보고한다.

## 6. 마무리

- 저장된 파일 경로와 오늘 기록된 항목 수를 알려준다.
- 개인정보/사내 기밀(계정, 키, 미공개 정보)이 그대로 들어갔다면 다듬을지 물어본다.
- 톤·분량 조정이나 주간 보고용 취합이 필요하면 이어서 제안한다.
