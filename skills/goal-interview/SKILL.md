# Goal Interview

사용자의 흐릿한 의도를 "참/거짓으로 판정 가능한 완료 기준"으로 바꾸는 것이 이
스킬의 유일한 목표다. 실제 산출물(코드·명세·문서 등) 생성은 이 스킬의 일이 아니며,
기준이 확정되면 다음 단계로 **넘긴다**(단일 책임). 핵심 철학: **사용자가 목표를
창조하게 만들지 말고, 반응하게 만든다.** 빈 페이지에 명세를 쓰는 것은 누구에게나
어렵지만, 선택지를 고르거나 위험한 가정을 바로잡는 "반응"은 쉽다.

이 스킬은 실시간 왕복 대화(인터뷰)이므로 **메인 에이전트에서 실행하며, 서브에이전트에
위임하지 않는다** — 위임하면 실행 중 사용자 입력을 받지 못해 인터뷰가 성립하지 않는다.

## 0. 발동 여부 확인

발동/비발동 판단은 frontmatter description을 따른다. 추가 기준 하나: 되돌리기 쉬운
사소한 작업(오타 수정 등)에는 쓰지 않는다 — 인터뷰 비용이 산출 가치보다 크다.

## 1. 절대 규칙

- **완료 기준이 확정되기 전에는 코드·파일 작업을 시작하지 않는다.**
- **인터뷰하는 쪽은 나(AI)다.** 한 번에 하나의 결정만 다룬다. 서로 무관한 질문을
  한꺼번에 쏟아내지 않는다 (사용자 피로가 실패의 주원인).
- **선택지·트레이드오프가 있는 결정**에는 `AskUserQuestion`으로 객관식을 제시한다.
  단순 확인("이게 맞나요?")이나 개방형 엣지 케이스 질문은 일반 대화로 묻되, 이때도
  후보 예시를 곁들여 사용자가 창조 대신 반응하게 한다. 한 번의 호출·질문은 하나의
  결정(또는 그에 딸린 밀접한 몇 개 필드)만 담는다.
- 추측으로 빈칸을 채우지 않는다. 여러 해석이 가능하면 조용히 하나 고르지 말고
  선택지로 펼쳐서 묻는다. (사용자가 판단을 위임하는 신호가 오면 §3-B를 따른다.)

## 2. 인터뷰 진행

아래를 순서대로 밟되, 사용자의 답에 따라 유연하게 오간다.

1. **의도 되짚기** — 사용자가 이루려는 것을 한 문장으로 요약해 "이게 맞나요?"로
   확인한다. 여기서부터 어긋나면 뒤가 전부 어긋난다.
2. **가정 노출 + 위험 표시** — 내가 이 작업에서 하려는 가정을 나열하고, 틀리면
   손해가 큰 것에만 ⚠️ 표시한다. **⚠️ 붙은 항목만 확인받는다** (나머지는 흘려보내
   피로를 줄인다). ⚠️ 가정에서 파생된 완료 기준은 ⚠️를 이어받는다.
3. **엣지 케이스로 구체화** — 애매하거나 실패할 수 있는 구체적 상황을 제시하고,
   각각 어떤 결과를 원하는지 묻는다. 사람은 추상적 기준("인증 정책은?")엔 막혀도
   구체적 사례("만료된 토큰이 오면 401? 403?")엔 즉답한다.
4. **모르면 선택지** — 사용자가 "모르겠다"고 하면, 그것은 아직 세상에 없는 결정을
   지금 만들어야 한다는 신호다. 흔한 선택지 2~3개를 **트레이드오프와 함께** 제시해
   객관식으로 바꿔준다.
5. **완료 미리보기 = 체크리스트 초안 검토** — 종료 전에 §3-C 형식의 체크리스트
   *초안*(해당되면 최종 PR 설명 포함)을 제시해 **한 번에** 검토받는다. 미결/가정의
   수용 여부도 이 검토에서 함께 확인한다 — 별도 검토 라운드를 만들지 않는다.
   상상 속 결과가 틀렸으면 한 줄도 짜기 전에 잡는다.

## 3. 종료 조건

카파시의 진단법을 적용한다: 주관적 질문("잘 됐나?")을 객관적 질문("이 조건이
참인가?")으로 바꿀 수 있으면 끝난 것이다.

- 정한 기준을 전부 **참/거짓으로 판정되는 문장**으로 다시 써본다.
- **판정 가능 ≠ 판정 방법을 안다.** 검증 요구는 위험도에 따라 차등 적용한다
  (고위험 = 되돌리기 어렵거나, 외부에 영향을 주거나, §4 승인 게이트 대상):
  - **고위험 기준**: 검증 수단에 더해 **기대 결과·합격선**까지 적는다 (명령이면
    예상 출력/종료코드, 성능이면 수치 임계값).
  - **저위험 기준**: `검증: 자명 — <판정 근거 한 구절>`로 줄여도 된다. 근거 구절
    없이 "자명"만 쓰는 것은 금지 — 분류가 보여야 사용자가 거부권을 행사한다.
  - 어느 쪽이든 참/거짓 문장 형식 자체는 유지한다. 검증을 못 적는 기준은 아직
    흐릿한 것이니 2번으로 돌아간다.
- 판정 불가한 항목이 남아 있으면 2번으로 돌아가 더 묻는다. 단, 아래 탈출 조건들을
  함께 적용해 무한 인터뷰를 막는다.
- **2회 백스톱** — 같은 항목을 2회 되물어도 좁혀지지 않으면 더 묻지 않고 합리적
  기본값을 골라 *미결/가정*으로 확정하고, 그 시점에 남은 미결 항목 수를 알려 끝이
  보이게 한다. **예외:** §4 승인 게이트 대상(비가역) 항목은 가정으로 확정하지
  않는다 — 대신 양자택일 질문 1회로 격상해 반드시 답을 받는다.

### 3-A. 수확체감 정지

남은 모호함이 **결과물을 실질적으로 바꾸지 못하면** 완벽을 좇지 말고 멈춘다. 그
항목은 판정 문장이 아니라 "가정"으로 명시해 체크리스트의 *미결/가정* 칸에 남긴다.
핵심 ⚠️ 위험과 판정 불가 항목이 해소됐으면 100% crisp하지 않아도 종료해도 된다.

### 3-B. 사용자 위임·피로 처리

명시적 위임("네가 정해")뿐 아니라 **암묵 신호도 위임으로 간주한다**: 단답·
"아무거나"류 답변이 2회 연속이면 사용자가 지친 것이다. 무한 질문 대신 **AI가
합리적 기본값을 골라** 진행하고, 그 선택은 *미결/가정* 칸에 근거와 함께 기록한다.
단, 되돌리기 어려운 것(§4 승인 게이트 대상)만은 기본값으로 밀지 말고 반드시
확인받는다.

### 3-C. 완료 기준 체크리스트 (최종 산출물 형식)

모든 기준이 판정 가능해지면 인터뷰를 끝내고 아래 형식으로 정리한다. §2-5에서 이미
초안을 검토받았으므로 여기서는 반영분만 확정한다. 이것이 핸드오프 대상이다.

```
## 완료 기준 체크리스트
- 목표: <이루려는 것 한 문장>
- 완료 기준:
  1. [참/거짓 판정 문장] — 검증: <명령 + 기대 결과/합격선> | <자명 — 판정 근거>
  2. ...
- 비목표(이번에 하지 않는 것): <스코프 크립 방지용 명시적 제외 항목>
- 미결/가정: <항목 + 선택 근거> — done 선언 전에 해소하거나 명시적으로 수용하고
  `(수용됨)`을 표기할 것. 침묵 속에 완료를 통과할 수 없다.
```

**비목표**는 "무엇이 done인가"만큼 중요하다 — 하지 않을 것을 못박아 스코프가
번지는 것을 막는다.

## 4. 마무리

- **CLAUDE.md 저축** — 앞으로도 반복될 만한 기준(코딩 컨벤션, 검증 명령 등)은
  CLAUDE.md에 넣을 문장으로 **제안만 하고 답변을 요구하지 않는다** (종료 시점에
  결정을 하나 더 얹지 않는다).
- **승인 게이트** — 위험하거나 되돌리기 어려운 단계(배포·PR 생성·삭제·외부 전송)
  앞에는 반드시 멈추고 사용자 승인을 받도록 계획에 명시한다.
- **핸드오프** — 완료 기준 체크리스트(§3-C)를 확정해 넘기는 것으로 이 스킬의
  역할은 끝난다. 실제 실행은 사용자 승인 후 **별도로** 진행하며, 필요하면 그 시점에
  적절한 에이전트(코드 구현이면 implementer 등)나 다른 스킬에 위임한다.
- **`/goal` 연계 제안** — 확정된 기준 중 **자동 판정 가능한 것**(테스트·명령
  실행으로 참/거짓이 갈리는 기준)이 있으면, 그 기준들만 모아 사용자가 그대로
  입력할 수 있는 `/goal <조건>` 한 줄을 만들어 제안한다 (Claude Code 내장
  커맨드, v2.1.139+ — 조건 충족까지 자동으로 턴을 반복). 슬래시 커맨드는
  사용자만 입력할 수 있으므로 **제안까지만** 한다. 자동 판정이 불가한 기준
  (육안 확인·주관 검토 등)은 `/goal` 줄에 넣지 말고 "완료 후 직접 확인" 목록으로
  분리해 함께 보여준다.
- **체크리스트 보존** — 핸드오프가 이 세션 밖(다음 세션·다른 에이전트)으로 넘어갈
  수 있으면 체크리스트를 **파일로도 남긴다**. 경로를 묻느라 사용자를 붙잡지 않는다
  — 프로젝트에 스펙 파일 관례가 있으면 그곳, 없으면 스크래치패드에 저장하고 위치만
  알린다. `(수용됨)` 표기 등 가정의 수용 여부는 반드시 이 파일 안에 기록한다 —
  구두로만 넘기면 다음 세션이 미결과 수용된 가정을 구별할 수 없다.
